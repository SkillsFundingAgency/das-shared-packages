# SFA.DAS.ApiContracts.Build

[![Build Status](https://sfa-gov-uk.visualstudio.com/Digital%20Apprenticeship%20Service/_apis/build/status%2Fdas-shared-packages-SFA.DAS.ApiContracts.Build?repoName=SkillsFundingAgency%2Fdas-shared-packages&branchName=master)](https://sfa-gov-uk.visualstudio.com/Digital%20Apprenticeship%20Service/_build/latest?definitionId=4310&repoName=SkillsFundingAgency%2Fdas-shared-packages&branchName=master)

[![NuGet Badge](https://buildstats.info/nuget/SFA.DAS.ApiContracts.Build)](https://www.nuget.org/packages/SFA.DAS.ApiContracts.Build/)

MSBuild tooling package that generates typed API contracts from an OpenAPI (Swagger) specification at build time, following SFA.DAS conventions. 
Produces `IGetApiRequest` / `IPostApiRequest` / `IPutApiRequest` / `IDeleteApiRequest` / `IPatchApiRequest` request wrappers, response DTOs, and 
an optional typed client for use with `SFA.DAS.Apim` and the outer apis.

## Pre-requisites

* An inner API project with build-time `swagger.json` generation configured via `Microsoft.Extensions.ApiDescription.Server`
* A contracts `.csproj` that references this package

## Quick Start

Add the package reference and point it at the inner API's `swagger.json`:

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <ApiContractsSwaggerJsonPath>$(MSBuildThisFileDirectory)..\MyApi\obj\swagger.json</ApiContractsSwaggerJsonPath>
    <ApiContractsNamespace>SFA.DAS.MyApi.Contracts</ApiContractsNamespace>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="SFA.DAS.ApiContracts.Build" Version="x.x.x" />
    <PackageReference Include="SFA.DAS.Apim.Shared" Version="x.x.x" />
  </ItemGroup>

</Project>
```

On the next build, two files are written to the `Generated/` folder alongside your `.csproj`:

| File | Contents |
|---|---|
| `Generated/Responses.g.cs` | Plain C# POCOs matching the shapes the inner API returns (generated by NSwag) |
| `Generated/Requests.g.cs` | Typed request wrappers implementing `IGetApiRequest`, `IPostApiRequest`, etc. |

Both files are automatically included in compilation and should be committed to source control.

## Configuration

All properties have defaults and are optional except `ApiContractsSwaggerJsonPath`.

| Property | Default | Description |
|---|---|---|
| `ApiContractsSwaggerJsonPath` | *(required)* | Path to the `swagger.json` produced by the inner API |
| `ApiContractsNamespace` | `$(RootNamespace).Generated` | Root namespace for all generated types |
| `ApiContractsOutputDirectory` | `$(MSBuildProjectDirectory)\Generated` | Directory where `.g.cs` files are written |
| `ApiContractsGenerationEnabled` | `true` | Set to `false` to skip generation entirely (e.g. in CI where files are already committed) |
| `ApiContractsClientName` | *(empty)* | When set, also generates a typed client — see [Client Generation](#client-generation) |
| `ApiContractsClientConfigurationNameOverride` | *(empty)* | Override the generated configuration class name — see [Configuration Name Override](#configuration-name-override) |

## Generated Requests

For each path and HTTP method in the swagger spec, the generator emits a class implementing the matching `SFA.DAS.Apim.Shared` interface. Each class is prefixed with an XML doc comment so that the method, path, and return type are visible on hover in the IDE.

Given this path in `swagger.json`:

```json
"paths": {
  "/api/vacancies/{vacancyId}": {
    "get": {
      "responses": {
        "200": {
          "content": {
            "application/json": {
              "schema": { "$ref": "#/components/schemas/VacancyResponse" }
            }
          }
        }
      }
    }
  }
}
```

The generator produces:

```csharp
/// <summary>GET /api/vacancies/{vacancyId} → <see cref="VacancyResponse"/></summary>
public record GetVacanciesByVacancyIdApiRequest(System.Guid VacancyId) : IGetApiRequest
{
    public string GetUrl => $"api/vacancies/{VacancyId}";
}
```

### Supported features

* **Path parameters** — become constructor arguments on the generated `record`
* **Query parameters** — required ones become constructor arguments; optional ones are nullable
* **Array query parameters** — serialised with repeated keys using `QueryHelpers`
* **Enum query parameters** — `$ref` enum types mapped to the generated enum name
* **Flags enums** — detected via `x-enumFlags: true`; serialised with spaces stripped
* **`DateTime` query parameters** — formatted as `"s"` (sortable ISO 8601)
* **POST / PUT / PATCH request bodies** — typed via the `Data` property; `$ref` and inline array bodies both supported
* **API versioning** — when `info.version` is anything other than `1.0`, a `Version` property override is emitted

## Client Generation

Setting `ApiContractsClientName` causes a third file, `Generated/Client.g.cs`, to be generated. This contains a configuration class, a typed interface, a concrete pass-through client, and a DI registration extension.

```xml
<PropertyGroup>
  <ApiContractsClientName>MyApi</ApiContractsClientName>
</PropertyGroup>
```

Produces:

```csharp
// Configuration
public class MyApiConfiguration : IInternalApiConfiguration
{
    public required string Url { get; set; }
    public required string Identifier { get; set; }
}

// Interface
public interface IMyApiClient<T> : IInternalApiClient<T>;

// Concrete client — delegates every method to the injected IInternalApiClient<T>
public class MyApiClient(IInternalApiClient<MyApiConfiguration> apiClient)
    : IMyApiClient<MyApiConfiguration> { ... }

// DI extension
public static class AddMyApiClientExtension
{
    public static IServiceCollection AddMyApiClient(
        this IServiceCollection services,
        IConfiguration configuration) { ... }
}
```

Register the client in your application's `Program.cs` or startup:

```csharp
services.AddMyApiClient(configuration);
```

The extension reads configuration from a section whose name matches the configuration class (`MyApiConfiguration` by default):

```json
{
  "MyApiConfiguration": {
    "Url": "https://my-inner-api.example.com",
    "Identifier": "https://my-inner-api.example.com"
  }
}
```

An `ArgumentException` is thrown at startup if the section is missing.

## Configuration Name Override

If the configuration section in your config store uses a name that differs from the generated default (`{ClientName}Configuration`), set `ApiContractsClientConfigurationNameOverride`:

```xml
<PropertyGroup>
  <ApiContractsClientName>MyApi</ApiContractsClientName>
  <ApiContractsClientConfigurationNameOverride>MyApiV2Configuration</ApiContractsClientConfigurationNameOverride>
</PropertyGroup>
```

The generated configuration class and all DI registration calls will use `MyApiV2Configuration` instead of `MyApiConfiguration`. The client interface and concrete class names are unaffected — they are always derived from `ApiContractsClientName`.

## Disabling Generation

To build a contracts project without running code generation (for example, in a CI pipeline where the generated files are already committed):

```xml
<PropertyGroup>
  <ApiContractsGenerationEnabled>false</ApiContractsGenerationEnabled>
</PropertyGroup>
```
